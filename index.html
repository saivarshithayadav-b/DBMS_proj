DATATYPES

Data Types are attributes that specify the type of data a column can hold in a database table.

🔹 Types of Data Types in DBMS

Numeric Data Types

Used to store numbers.
INT / INTEGER → Whole numbers
Example: 10, 200
FLOAT / REAL → Decimal numbers
Example: 10.5, 3.14
DECIMAL(p, s) → Fixed precision numbers
Example: 99.99

2. Character (String) Data Types

Used to store text.
CHAR(n) → Fixed-length string
Example: CHAR(5) → "HELLO"
VARCHAR(n) → Variable-length string
Example: VARCHAR(50) → "Varshitha"
TEXT → Large text data

3. Date and Time Data Types

Used to store date and time values.
DATE → Stores date (YYYY-MM-DD)
TIME → Stores time (HH:MM:SS)
DATETIME / TIMESTAMP → Stores both date and time

4. Boolean Data Type

Stores TRUE or FALSE values

Binary Data Types

Used to store images, files, etc.
BLOB (Binary Large Object)
BINARY / VARBINARY

DDL COMMANDS (Data Definition Language)

DDL commands are used to define, create, modify and drop database structures.

1. CREATE - Create new database/table

CREATE DATABASE netflix_db;

CREATE TABLE users (
  user_id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL,
  email VARCHAR(100) UNIQUE,
  created_date DATE
);

2. ALTER - Modify table structure

ALTER TABLE users ADD COLUMN age INT;
ALTER TABLE users DROP COLUMN age;
ALTER TABLE users MODIFY email VARCHAR(150);

3. DROP - Delete database/table

DROP TABLE users;
DROP DATABASE netflix_db;

4. TRUNCATE - Remove all records (keeps structure)

TRUNCATE TABLE users;

5. RENAME - Rename table

RENAME TABLE users TO customers;

CONSTRAINTS

Constraints are rules applied to columns/tables to ensure data integrity and validity.

1. NOT NULL - Column must have a value

CREATE TABLE movies (
  movie_id INT,
  title VARCHAR(100) NOT NULL,
  director VARCHAR(50) NOT NULL
);

2. UNIQUE - All values must be unique

CREATE TABLE users (
  user_id INT,
  email VARCHAR(100) UNIQUE
);

3. PRIMARY KEY - Unique identifier for records

CREATE TABLE users (
  user_id INT PRIMARY KEY,
  username VARCHAR(50)
);

4. FOREIGN KEY - Link to primary key in another table

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

5. CHECK - Values must satisfy condition

CREATE TABLE movies (
  movie_id INT,
  rating DECIMAL(3,1) CHECK (rating >= 0 AND rating <= 10)
);

6. DEFAULT - Set default value

CREATE TABLE downloads (
  download_id INT,
  download_date DATE DEFAULT CURDATE()
);

DML COMMANDS (Data Manipulation Language)

DML commands are used to manipulate data in the database.

1. INSERT - Add new records

INSERT INTO users (user_id, username, email)
VALUES (1, 'john_doe', 'john@example.com');

INSERT INTO users VALUES (2, 'jane_smith', 'jane@example.com');

2. UPDATE - Modify existing records

UPDATE users SET email = 'newemail@example.com' WHERE user_id = 1;

UPDATE movies SET rating = 8.5 WHERE movie_id = 5;

3. DELETE - Remove records

DELETE FROM users WHERE user_id = 1;

DELETE FROM orders WHERE order_date < '2020-01-01';

4. SELECT - Retrieve data

SELECT * FROM users;

SELECT username, email FROM users WHERE user_id > 5;

BASIC SQL COMMANDS

1. SELECT - Retrieve columns

SELECT username, email FROM users;

2. WHERE - Filter records

SELECT * FROM movies WHERE rating > 7.5;

3. ORDER BY - Sort results

SELECT * FROM movies ORDER BY rating DESC;
SELECT * FROM users ORDER BY username ASC;

4. LIMIT - Restrict number of results

SELECT * FROM movies LIMIT 10;
SELECT * FROM users LIMIT 5 OFFSET 10;

5. DISTINCT - Remove duplicates

SELECT DISTINCT genre FROM movies;
SELECT DISTINCT country FROM users;

6. BETWEEN - Range condition

SELECT * FROM movies WHERE rating BETWEEN 7 AND 9;

7. IN - Multiple conditions

SELECT * FROM users WHERE country IN ('USA', 'UK', 'Canada');

8. LIKE - Pattern matching

SELECT * FROM users WHERE username LIKE 'john%';
SELECT * FROM movies WHERE title LIKE '%Marvel%';

JOINS - Combine data from multiple tables

1. INNER JOIN - Returns matching records from both tables

SELECT users.username, orders.order_id
FROM users
INNER JOIN orders ON users.user_id = orders.user_id;

2. LEFT JOIN (LEFT OUTER JOIN) - All records from left + matching from right

SELECT users.username, orders.order_id
FROM users
LEFT JOIN orders ON users.user_id = orders.user_id;

3. RIGHT JOIN (RIGHT OUTER JOIN) - All records from right + matching from left

SELECT users.username, orders.order_id
FROM users
RIGHT JOIN orders ON users.user_id = orders.user_id;

4. FULL OUTER JOIN - All records from both tables

SELECT users.username, orders.order_id
FROM users
FULL OUTER JOIN orders ON users.user_id = orders.user_id;

5. CROSS JOIN - Cartesian product of both tables

SELECT users.username, movies.title
FROM users
CROSS JOIN movies;

6. SELF JOIN - Join table with itself

SELECT a.user_id, a.username, b.username
FROM users a, users b
WHERE a.referrer_id = b.user_id;

AGGREGATE OPERATORS

Aggregate functions perform calculations on multiple rows and return a single value.

1. COUNT() - Count records

SELECT COUNT(*) FROM users;
SELECT COUNT(DISTINCT country) FROM users;

2. SUM() - Sum values

SELECT SUM(price) FROM orders;
SELECT category, SUM(amount) FROM purchases GROUP BY category;

3. AVG() - Average value

SELECT AVG(rating) FROM movies;
SELECT genre, AVG(rating) FROM movies GROUP BY genre;

4. MAX() - Maximum value

SELECT MAX(price) FROM products;
SELECT category, MAX(salary) FROM employees GROUP BY category;

5. MIN() - Minimum value

SELECT MIN(price) FROM products;
SELECT department, MIN(salary) FROM employees GROUP BY department;

6. GROUP BY - Group rows by columns

SELECT genre, COUNT(*) FROM movies GROUP BY genre;
SELECT user_id, COUNT(order_id) FROM orders GROUP BY user_id;

7. HAVING - Filter aggregated results

SELECT genre, COUNT(*) FROM movies GROUP BY genre HAVING COUNT(*) > 5;
SELECT department, AVG(salary) FROM employees GROUP BY department HAVING AVG(salary) > 50000;

SUB QUERIES (Nested Queries)

A query within another query to retrieve data based on complex conditions.

1. Subquery in WHERE Clause

SELECT * FROM movies WHERE rating > (SELECT AVG(rating) FROM movies);

SELECT * FROM users WHERE user_id IN (SELECT user_id FROM orders WHERE total > 100);

2. Subquery in FROM Clause

SELECT * FROM (SELECT user_id, COUNT(*) as order_count FROM orders GROUP BY user_id) AS user_orders WHERE order_count > 5;

3. Correlated Subquery

SELECT u.username, (SELECT COUNT(*) FROM orders o WHERE o.user_id = u.user_id) as total_orders FROM users u;

4. EXISTS Operator

SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.user_id);

5. IN Operator with Subquery

SELECT * FROM movies WHERE genre IN (SELECT DISTINCT genre FROM top_rated_movies WHERE rating >= 9);

6. Comparison with Subquery

SELECT * FROM products WHERE price = (SELECT MAX(price) FROM products);
SELECT * FROM employees WHERE salary > (SELECT AVG(salary) FROM employees);

VIEWS

A view is a virtual table created from a SQL query. It doesn't store data but displays data from tables.

1. CREATE VIEW - Create a virtual table

CREATE VIEW user_orders_view AS
SELECT u.username, o.order_id, o.order_date, o.total
FROM users u
INNER JOIN orders o ON u.user_id = o.user_id;

2. Using Views - Query the view like a table

SELECT * FROM user_orders_view WHERE total > 500;

3. CREATE VIEW with WHERE

CREATE VIEW high_rated_movies AS
SELECT * FROM movies WHERE rating >= 8.0;

4. ALTER VIEW - Modify a view

ALTER VIEW high_rated_movies AS
SELECT * FROM movies WHERE rating >= 8.5;

5. DROP VIEW - Delete a view

DROP VIEW user_orders_view;

Advantages of Views:

  • Simplifies complex queries
  • Provides security by hiding table structures
  • Provides data independence
  • Reduces redundancy

RELATIONAL ALGEBRA - UNARY OPERATIONS

Unary operations work on a single relation/table.

1. PROJECTION (π) - Select specific columns

πusername, email(users)

SQL Equivalent:
SELECT username, email FROM users;

2. SELECTION (σ) - Select rows based on condition

σrating > 7.5(movies)

SQL Equivalent:
SELECT * FROM movies WHERE rating > 7.5;

3. RENAME (ρ) - Rename relations/attributes

ρnew_name(old_name)

SQL Equivalent:
SELECT * FROM users AS customers;

Combined Example:

πusername, rating(σrating > 7(movies))

SQL Equivalent:
SELECT username, rating FROM movies WHERE rating > 7;

RELATIONAL ALGEBRA - BINARY OPERATIONS

Binary operations work on two relations/tables.

1. UNION (∪) - Combine and remove duplicates

users ∪ customers

SQL Equivalent:
SELECT * FROM users
UNION
SELECT * FROM customers;

2. INTERSECTION (∩) - Common rows in both

users ∩ premium_users

SQL Equivalent:
SELECT * FROM users
INTERSECT
SELECT * FROM premium_users;

3. SET DIFFERENCE (-) - Rows in first but not second

users - inactive_users

SQL Equivalent:
SELECT * FROM users
EXCEPT
SELECT * FROM inactive_users;

4. CARTESIAN PRODUCT (×) - All combinations

users × movies

SQL Equivalent:
SELECT * FROM users
CROSS JOIN movies;

5. JOIN (⋈) - Combine tables on condition

users ⋈ orders

SQL Equivalent:
SELECT * FROM users
INNER JOIN orders ON users.user_id = orders.user_id;

6. DIVISION (/) - Advanced operation

Find users who have watched all action movies

SQL Equivalent (using NOT EXISTS):
SELECT DISTINCT u.user_id FROM users u
WHERE NOT EXISTS (SELECT 1 FROM action_movies am
WHERE NOT EXISTS (SELECT 1 FROM watched w
WHERE w.user_id = u.user_id AND w.movie_id = am.movie_id));
